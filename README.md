# 设计模式
本项目主要实现了《Design Patterns in Modern C++20》一书内的设计模式，并使用C++20进行实现。

# 模式目录
## 01.创建型模式 Builder
相关文件: 01_Builder.cpp, 01_Builder.hpp  
构造器模式Builder主要是用来创建复杂对象，通过将复杂对象的创建过程与表示分离，使得同样的构建过程可以创建不同的表示，最终目的是通过流式接口调用链来简化构造过程。为了实现流式接口，需要构造器的函数返回this或者*this。 
原则上所有参数都可以显示使用单独函数进行设置，但这样会导致代码冗余，因此使用Builder模式来简化构造过程。 

## 02.工厂模式 Factory
相关文件: 02_Factory.cpp, 02_Factory.hpp  
工厂模式Factory主要是把使用可以返回多态对象的函数，来避免因为层次的增加而增量修改所有调用方的函数。通过把条件限制在工厂内部和返回多态对象的定义，可以限制调用方对非法对象的访问。 
一般会返回一个指针，因为指针可以指向基类，而基类指针可以指向派生类，从而实现多态。 

## 03.适配器模式 Adapter
相关文件: 03_Adapter.cpp, 03_Adapter.hpp   
适配器模式Adapter主要是用来把一个接口转换成另一个接口，使得原本不兼容的类可以一起工作。  
适配器本质上是把转换的代价转嫁到适配器上，从而隔离两个接口间的直接依赖关系。  

## 04.代理模式 Proxy
相关文件: 04_Proxy.cpp, 04_Proxy.hpp   
代理模式Proxy主要是用来控制对对象的访问，强化现有成员的潜在行为，通过代理对象来间接访问目标对象，从而可以在访问目标对象之前进行一些额外的操作。  
使用代理模式的前提是:被代理的对象不能够直接访问，或者需要控制对被代理对象的访问。  

## 05.职责链模式 Chain of Responsibility
相关文件: 05_CoR.cpp 05_CoR.hpp  
职责链模式Chain of Responsibility主要是让组件依次处理命令(或查询)，直到命令被处理或者所有组件都被遍历。本质上是维护一个单链，如果有必要也可以使用普通的vector或者list来代替指针链。  

## 06.中介者模式 Mediator
相关文件: 06_Mediator.cpp 06_Mediator.hpp
中介者模式Mediator主要是用来把组件间的直接依赖关系解除，通过中介者来管理组件间的交互，从而使得组件之间解耦。  
当组件A/组件B/组件C之间需要交互时，全部通过一个组件Z来完成，组件A/组件B/组件C之间不再直接交互，从而实现解耦。
